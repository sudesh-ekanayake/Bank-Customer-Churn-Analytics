📌 Project Overview

This project uses Data Mining and Machine Learning techniques to predict customer churn in the banking sector. The goal is to identify at-risk customers and provide data-driven insights to improve retention rates

🛠️ Tech Stack & Methods

Data Analysis: Python (Pandas, Numpy, Seaborn, Matplotlib).
Machine Learning:
           Random Forest (Best Performance: 95.6% Accuracy).
           Decision Tree & Logistic Regression.
Clustering: K-Means Clustering for customer segmentation into 3 distinct groups.
Visualization: Interactive 3-page Power BI Dashboard.

📂 Repository Structure
    📁 Notebooks: Contains the Python Jupyter Notebook with code for cleaning and modeling.
    📁 Data: Includes the Credit Card Customers dataset.
    📁 PowerBI: The .pbix file for interactive churn risk analytics.
    📁 Documentation: Presentation slides and project methodology.
📈 Key Insights from Analytics
    Top Churn Driver: Total Transaction Count is the most significant indicator. A drop in transactions signals a high risk of churn.
    Inactivity Risk: Customers inactive for more than 2 months are significantly more likely to leave the bank.
    Card Tiers: Platinum and Gold cardholders show a higher churn tendency compared to Blue cardholders.
